BiwengerProManager API

REST wrapper for Biwenger data — fast, simple and designed for integrations and football analytics tools.

Overview

This API exposes leagues, players, rounds and premium endpoints (transfers and clauses). It is suitable for dashboards, bots and football analytics tools.

Kickoff: JSON-ready data and a project layout designed for local development and production deployment (DocumentRoot = public/).
  • Main routes: /api/v0 (free), /api/v1 (premium)
  • Structure: repositories, services and controllers live in src/ (not web-accessible)
  • Storage: MongoDB (configured via environment variables or config/{env}/.env)

How to run locally

Quick options — choose depending on your platform. The configuration loader reads the single root configuration file at config/.env (if present). You can also export environment variables directly in your shell.

  1. Edit config/.env with your credentials and set APP_ENV=local. Use keys such as MONGODB_URI, MONGODB_DB, API_KEY, CONFIG_SECRET, BOT_EMAIL, BOT_PASSWORD.
  2. Linux/macOS: use the included launcher: ./scripts/run.sh (it loads config/.env and starts PHP built-in server).
  3. Windows: start Apache from XAMPP Control Panel (DocumentRoot → public/) or use WSL and run ./scripts/run.sh. Alternatively run the PHP built-in server directly: php -S localhost:8000 -t public.
  4. There is also ./scripts/setup-unix.sh for initial environment checks on Unix-like systems.

After starting, open http://localhost:8000/ (or replace host/port if you changed APP_PORT/APP_HOST).

Quick examples

Request leagues or players directly:

GET /api/v0/leagues
GET /api/v0/players/{playerId}

From the terminal:

curl http://localhost:8000/api/v0/leagues

Deployment tips

In production use Apache/Nginx with DocumentRoot pointing to public/. Never expose the src/ folder.

If you use a VirtualHost and want a different port, add Listen <PORT> to Apache and configure a matching <VirtualHost *:<PORT>> block.
